## 一、最高宪法：6 个边界法则

### 1. 螺旋的终止条件

- **量化拐点**：连续两圈在"元回顾"时发现，新搜到的工具评分（skill-judge）均低于现有武库的平均分。说明开源生态红利已被榨干。
- **实战拐点**：当你交给我一个真实的复杂业务需求时，我一次都没有因为"缺少某个信息获取、代码执行或状态同步的能力"而卡住。此时螺旋暂停，从"武备期"全面转入"实战期"。

### 2. 回顾环节结构化

- 绝对禁止漫谈。每圈结束时，必须由主 Agent 强制生成 **《Loop_X_MetaReview.md》**，强制回答 5 个灵魂拷问，并**必须产出至少 1 条针对下一圈 SOP 的修改补丁**。

### 3. 多 Agent 启用时机

- **Loop 1：绝对不用。** 先用单机跑通单圈闭环，巩固 SOP。
- **Loop 2：局部引入。** 仅在"广度搜索"环节，派 2 个 Teammate 并发搜不同渠道，主 Agent 汇总。
- **Loop 3+：全面流水线。** 搜（Researcher）、评（Evaluator）、装与文档（Installer）角色分离。

### 4. 自造 Skill 优先级

- **"15 分钟放弃原则"（Buy first, Build later）。** 多路并发搜了 15 分钟，找不到 >80 分的工具，立刻触发 `skill-creator` 自己造。

### 5. MCP vs Skill 选择标准

- **选 MCP（感官与通道）**：涉及读取外部状态、连接持久化数据源、调用外网 API。
- **选 Skill（大脑与肌肉）**：涉及无状态逻辑处理、本地工作流封装、Prompt 思考范式。

### 6. 夜间自主运行边界

- **绿灯（放权自主跑）**：所有"只读（Read）与计算"操作。搜索、打分、起草建议、写规划。
- **红灯（必须卡点等 Y）**：所有"写入（Write）与系统修改"操作。修改全局环境、执行未验证安装脚本、修改核心 CLAUDE.md。

## 二、单圈标准作业流 (SOP)

| 阶段 | 核心动作与强制纪律 | 解决的痛点 |
| --- | --- | --- |
| **1. 锚定 (Scope)** | 明确本圈唯一的主题 | 避免无头苍蝇式乱逛 |
| **2. 发散 (Hunt)** | 禁用单渠道。强制组合调用 skills find + context7 + GitHub | 搜索渠道单一 |
| **3. 审判 (Judge)** | 强制量化。所有候选者必须过 skill-judge。低于 80 分直接淘汰，记录淘汰原因防重复 | 评估没标准化 |
| **4. 试装 (Equip)** | 强制纪律。触发 verification-before-completion，写冒烟测试，跑通才算装上 | 安装不够稳 |
| **5. 织网 (Weave)** | 更新 ARMORY.md（武器库索引），并写入触发规则 | 装了不触发、知识没累积 |
| **6. 元回顾 (Meta)** | 强制填写结构化复盘报告，生成下一圈的 SOP 补丁 | 没有元迭代 |

## 三、螺旋演进路线图

### Loop 1：基建与扫盲（Bootstrap）
- 目标：治好系统"健忘"和"无纪律"
- 动作：解决跨会话记忆；配置触发规则让已有 Skill 被自动调用；单 Agent 跑通完整 6 步 SOP

### Loop 2：感官扩张
- 目标：引爆信息获取能力
- 动作：引入高级浏览器工具、深度 GitHub 爬取；搜索阶段启用多 Agent 并发

### Loop 3：生态缝合
- 目标：填补高阶空白
- 动作：大规模接入 Anthropic MCP Registry；启动 skill-creator 批量自造特种工具
