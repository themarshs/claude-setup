# Security Gate（预提交安全清单）

> 来源：社区 3 仓库共识（48.6K + 13.2K + 6.8K stars）提炼

## 强制检查清单（每次提交前）

- [ ] 无硬编码密钥（API keys、passwords、tokens、connection strings）
- [ ] 所有用户输入已验证（边界处用 Zod/schema，快速失败）
- [ ] SQL 使用参数化查询（禁止字符串拼接）
- [ ] HTML 输出已转义（防 XSS）
- [ ] 错误消息不泄露内部实现（堆栈、路径、版本号）
- [ ] 认证/授权逻辑已验证（JWT 过期、权限检查）
- [ ] 新依赖已审计（`npm audit` / `pip audit` / `gosec`）
- [ ] 敏感数据不写入日志

## 密钥管理

- 环境变量或密钥管理器，永不硬编码
- 启动时验证必需密钥存在（缺失立即 fatal）
- 疑似泄露 → 立即轮换，不等修复

## 发现安全问题时

1. 停止当前工作
2. 评估影响范围（单文件 / 跨模块 / 全系统）
3. 修复关键问题后再继续
4. 检查代码库是否有同类模式
