# 配置变更纪律（Config-Before-Guess Rule）

## 核心规则

遇到环境、配置、工具链问题时：

1. **先查文档，后改文件** — 用 context7 / 官方 docs / WebSearch 确认正确做法
2. **禁止凭记忆改配置** — 配置格式、注册方式、API 参数会变，训练数据不可靠
3. **同类方法只算一次 Strike** — 改不同文件但方法相同（都是手动编辑 JSON）= 同一次尝试，不重置 Strike 计数

## 判定标准

以下情况全部属于"同一种方法"，不算换方法：
- 改 `.mcp.json` vs 改 `settings.local.json` vs 改 `settings.json` → 都是"手动编辑配置文件"
- 改环境变量 vs 改 .env 文件 → 都是"修改环境变量来源"
- 换绝对路径 vs 换相对路径 → 都是"调整命令路径"

真正的"换方法"：
- 手动编辑文件 → 用 CLI 命令注册
- 改配置 → 查文档确认正确配置方式
- 本地调试 → 查社区/Issue 看别人怎么解决

## 触发条件

任何涉及以下场景时自动激活：
- MCP Server 注册/连接问题
- CLI 工具配置
- 环境变量/密钥管理
- 构建/部署配置
