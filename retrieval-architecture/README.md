---

## 设计原则

1. **先检查手上有什么工具 → 用对应工具检索 → 再输出**
2. **每个 Teammate 只装它需要的 Skills（token 成本约束）**
3. **Skills 是方法论文档，不是代码——教 agent 什么时候调用、怎么调用**
4. **先搞懂原理再装，不盲装**
